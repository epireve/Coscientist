# MCP servers index

Custom Model Context Protocol servers shipped by Coscientist.
Auto-generated by `lib/mcp_index.py`. Regenerate with:

```bash
uv run python -m lib.mcp_index > MCP_SERVERS.md
```

A test (`tests/test_mcp_index.py`) asserts this file matches the
generator output, so a stale `MCP_SERVERS.md` will fail CI.

## Install

```bash
/plugin marketplace add epireve/coscientist
/plugin install <plugin-name>@coscientist
```

## Per-server detail

### `coscientist-graph-query-mcp` (0.1.0)

Read-only stdio MCP over Coscientist's per-project citation / concept / author graph. Wraps lib/graph.py SQLite-adjacency tables and adds BFS shortest-path. Forward-compatible with the planned Kuzu backend migration.

- **Server name** (in MCP host): `graph-query`
- **Plugin path**: `plugin/coscientist-graph-query-mcp`
- **Tools**: `neighbors`, `walk`, `in_degree`, `hubs`, `node_info`, `shortest_path`
- **Keywords**: `graph`, `citation-graph`, `kuzu`, `sqlite`, `research`, `mcp`, `read-only`

### `coscientist-manuscript-mcp` (0.1.0)

Stdio MCP server: .docx / .tex / .md → structured AST. Extracts sections, citations (latex / pandoc / numeric / author-year), word count. Markdown + LaTeX paths are pure stdlib; .docx requires pandoc on PATH.

- **Server name** (in MCP host): `manuscript`
- **Plugin path**: `plugin/coscientist-manuscript-mcp`
- **Tools**: `detect_format`, `extract_citations`, `extract_sections`, `parse_manuscript`
- **Keywords**: `manuscript`, `ast`, `citations`, `latex`, `markdown`, `docx`, `mcp`

### `coscientist-retraction-mcp` (0.1.0)

Stdio MCP server for retraction status + PubPeer comments. Wraps Crossref's update-to / updated-by fields and PubPeer's public publication API. Pure-stdlib networking; no API keys required.

- **Server name** (in MCP host): `retraction`
- **Plugin path**: `plugin/coscientist-retraction-mcp`
- **Tools**: `lookup_doi`, `batch_lookup`, `pubpeer_comments`
- **Keywords**: `retraction`, `retraction-watch`, `pubpeer`, `crossref`, `mcp`, `research-integrity`

**Total servers**: 3
